Infrastructure: AWS Fargate & ALB

af-south-1 (Cape Town)
Serverless Compute
All 8 microservices run on Amazon ECS using AWS Fargate, eliminating the need to manage underlying EC2 instances.
High availability across multiple AZs
Automated scaling based on load
Isolated container environments
Intelligent Routing
An Application Load Balancer (ALB) serves as the single entry point, routing traffic to specific services based on URL paths.
Path-based routing (e.g., /api/v1/auth)
Health checks and automatic failover
Centralized SSL termination
Rapid Hot-Patching
To bypass sandbox ECR upload restrictions, a custom S3-based patching mechanism is used for rapid route updates.
Patches uploaded to S3 bucket
Init script fetches patches on startup
Enables fixes without full image rebuilds
